How it works

1
Claude calls MCP tools. The plugin exposes ensure_endpoint_up, run_shell, reset_session, and stop_endpoint as first-class tools inside Claude Code.
2
Globus Compute bridges to the facility. A personal endpoint on Anvil receives the work, allocates a Slurm block, and keeps it warm between calls.
3
A persistent session shell runs on the node. Working directory and environment carry across run_shell calls — so writing a file, installing a package, and running it all share one live context on a006.
4
Cost governance is built in. Each response reports session_spend; stop_endpoint releases the block the moment work finishes.
Note (POC): the endpoint reported a sandboxing warning on every call ("Task sandboxing will not work due to endpoint misconfiguration") — tasks ran unsandboxed. Worth enabling sandboxing on the endpoint before this moves past proof-of-concept.

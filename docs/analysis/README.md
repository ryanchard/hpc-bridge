# hpc-bridge — Deep Analysis

A multi-perspective, **steelman + de-risk** analysis of the [hpc-bridge vision](../vision.md):
should it be built, how should it actually work, and what has to be true for it to win.

The verdict is **BUILD WITH CONDITIONS**. The engineering core (a personal Globus Compute
endpoint giving an agent a warm, low-latency foothold on real HPC) is sound and genuinely
differentiated. But **7 of the vision's 10 load-bearing claims were refuted** under adversarial
verification — almost all by over-extending a true narrow premise ("runs as your UID, no root,
no identity-mapping daemon") into a false absolute ("zero new attack surface," "no new review,"
"survives transparently," "credentials effectively closed," "Globus Compute is the moat,"
"accumulate then reveal"). The honest, narrower version of the pitch is **stronger** than the
vision's: NERSC-first, scale-to-zero-by-default, facility-co-designed, with the credential
boundary and cost governance enforced as runtime control planes rather than asserted.

## How this was produced

Generated by a ~50-agent workflow: 5 web-grounded research streams → an 8-persona debate
(facility CISO, sysadmin, Globus Compute architect, agent-UX, credential-security, allocation PI,
GTM strategist, kill-the-project skeptic) → a dedicated interactive-SSH/MFA design track →
adversarial verification of the 10 load-bearing claims (3 lenses each, majority vote) → synthesis.

**Every facility fact and external citation was independently fact-checked against primary
sources** (NERSC/ALCF/OLCF/TACC docs, the MCP spec, Globus Compute docs, the cited arXiv papers).
Overstated statistics surfaced by that check were corrected or hedged before commit. Items still
marked `[low-confidence]` / `[med]` inline are flagged precisely because they could **not** be
confirmed and must not be cited as settled facts.

## The 10 load-bearing claims, verified

| Claim | Verdict |
|-------|---------|
| C1 — Trust boundary "identical to SSH / zero new attack surface / no new review" | **REFUTED** |
| C2 — Persistence stops mattering; endpoints survive death transparently | **REFUTED** |
| C3 — Non-interactive SSH under facility MFA works across leadership facilities | **REFUTED** (holds at NERSC only) |
| C4 — Globus Compute is a durable moat no competitor can match | **REFUTED** (the moat is Globus Auth, and it's shared) |
| C5 — Personal-endpoint flywheel manufactures MEP demand | **REFUTED** |
| C6 — Filesystem-as-state is sufficient for v1 | STANDS-WITH-CAVEATS |
| C7 — Per-facility config solvable via templates + probing | STANDS-WITH-CAVEATS |
| C8 — Persistent login-node coordinator is tolerable | **REFUTED** (sanctioned-mechanism-only, not "because it's lightweight") |
| C9 — Credentials kept out of the LLM via separate process + scrub hook | **REFUTED** (needs separate principal + credential-free hot path) |
| C10 — Warm-pilot delivers seconds-scale interactive loop | STANDS-WITH-CAVEATS (real but unmeasured on a real node; warm-only) |

## The four highest-leverage changes (if you read nothing else)

1. **Delete the dangerous sentence.** Replace "identical to SSH / zero new attack surface / no
   new review" (C1) with: *identical **privilege** boundary to SSH (your UID, no root, no
   identity-mapping daemon), **plus** a new agent command channel we make facility-observable,
   facility-revocable, and periodically re-MFA'd.* This converts the weakest claim into the security pitch.
2. **Invert the GTM (C5).** "Accumulate shadow endpoints, then reveal telemetry" → "co-design with
   one named facility (NERSC) from day zero; gate launch on one on-record facility reference."
3. **Scale-to-zero is the only default; warm is a budget-gated, TTL-capped, logged opt-in (C10/cost).**
   Enforce cost and credentials as runtime control planes in the MCP server, not as documentation.
4. **Run the kill-first experiments before packaging (04 §7).** E1 (measured warm RTT on a real
   Perlmutter node), E3 (broker home exists), E4 (scheduler-safe self-heal) can each invalidate the
   thesis in under a week — do them first.
